Práctica 5 Computación en la Nube

1. Subida Archivos.

1.1 Imágenes.

Elemento Descripción
Descripción Método que permite cargar una Imagen.
URL /api/Upload/Img
Devuelve
  • fieldname: Key de la petición POST.
  • originalname: Nombre original del archivo.
  • encoding: Tipoi de codificación.
  • mimetype: Tipo de archivo(text/jpeg,...).
  • destination: Lugar donde se va a almacenar.
  • filename: Nueno nombre del fichero. Este deberá ser utilizado posteriormente.
  • path: Lugar de Almacenamiento.
  • size: Tamaño del archivo.

1.2 Hojas de Cálculo.

Elemento Descripción
Descripción Método que permite cargar un archivo de tipo Hoja de Cálculo.
URL /api/Upload/HojaCalculo
Devuelve
  • fieldname: Key de la petición POST.
  • originalname: Nombre original del archivo.
  • encoding: Tipoi de codificación.
  • mimetype: Tipo de archivo(text/csv,...).
  • destination: Lugar donde se va a almacenar.
  • filename: Nueno nombre del fichero. Este deberá ser utilizado posteriormente.
  • path: Lugar de Almacenamiento.
  • size: Tamaño del archivo.

2. Recuperar Archivos.

2.1 Imágenes.

2.1.1 Originales.

Elemento Descripción
Descripción Método que permite recoger una Imagen subida previamente.
URL /api/Get/Img/Original/:name
Parámetros
  • Name: Nombre de la Imagen a recoger.

2.1.2 Procesadas.

Elemento Descripción
Descripción Método que permite recoger una Imagen procesada previamente.
URL /api/Get/Img/Final/:name
Parámetros
  • Name: Nombre de la Imagen a recoger.

2.2 Hojas de Cálculo.

2.2.1 Originales.

Elemento Descripción
Descripción Método que permite recoger una Imagen subida previamente.
URL /api/Get/HojaCalculo/Original/:name
Parámetros
  • Name: Nombre de la HojaCalculo a recoger.

2.2.2 Procesadas.

Elemento Descripción
Descripción Método que permite recoger una Imagen subida previamente.
URL /api/Get/Img/HojaCalculo/Final/:name
Parámetros
  • Name: Nombre de la Gráfica a recoger (Imagen).

3. Filtrado Imágenes.

3.1 Filtrado Secuencial.

Elemento Descripción
Descripción Método que permite aplicar un filtrado a una imagen subida previamente. Este filtrado usa un algoritmo secuencial.
URL /api/Secuencial/:name
Parámetros
  • Name: Nombre del fichero previamente subido.
Nuevo Nombre Secuencial-Name

3.2 Filtrado con OpenMp.

Elemento Descripción
Descripción Método que permite aplicar un filtrado a una imagen subida previamente. Este filtrado usa un algoritmo paralelo.
URL /api/OpenMP/:name
Parámetros
  • Name: Nombre del fichero previamente subido.
Nuevo Nombre OpenMP-Name

3.3 Filtrado con MPI.

Elemento Descripción
Descripción Método que permite aplicar un filtrado a una imagen subida previamente. Este filtrado usa un algoritmo paralelo.
URL /api/MPI/:name/:height/:width
Parámetros
  • name: Nombre del fichero previamente subido.
  • height: Altura de la imagen.
  • height: Anchura de la imagen.
Nuevo Nombre MPI-Name

4. Gráficas.

Elemento Descripción
Descripción Método que permite aplicar un filtrado a una imagen subida previamente. Este filtrado usa un algoritmo paralelo.
URL /api/Grafica/:dataEntry/:elementoX/:elementoY/:tipoRepresentacion/:tipoGrafica
Parámetros
  • dataEntry: Nombre del fichero previamente subido.
  • elementoX: Elemento del eje X.
  • elementoY: Elemento del eje Y.
  • tipoRepresentacion: En caso de agrupar el tipo de plateamiento que se quiere realizar.
    1. Suma.
    2. Máximo.
    3. Mínimo.
  • tipoGrafica: Indicar el tipo de gráfica a devolver.
    1. Gráfica de Líneas.
    2. Gráfica de Barras.
    3. Gráfica de puntos.
    4. Resumen.
Nuevo Nombre
  • Gráfica de Líneas: Lineas-dataEntry.
  • Gráfica de Barras: Barras-dataEntry.
  • Gráfica de Puntos: Puntos-dataEntry.
  • Resumen: Resumen-dataEntry.
